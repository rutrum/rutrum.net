Curly

I never liked writing these static webpages because HTML has got to be one of the most verbose and bloated markup languages. It is an awful format so I've set out to try and simplify it.

Enter Curly, a markup language for HTML with a simplified format similar to JSON and CSS. I've managed to implement an abbreviated syntax that can be converted into HTML. The documentation is very limited, and because I've implemented the same translator in multiple languages (each in a different stage of progression) the syntax is open to change!

The current ambissions are to define prebuilt variables in HTML that can be reference in a single page, or across an entire project. Similarly, defining tag structures that also can be reference anywhere and generated uniquely given parameters. I still haven't determined a syntax in order to accomplish this, so the project has been on the back burner since.

I'm still very convinced that this is a good alternative to HTML, and has unique features against other HTML replacements such as HAML and pug. I look forward to really solidifying the grammar and just picking a language that can both handle file handling well and also easy to write object syntax (for trees!)
